# 个人经验与教训
1. 网络大小与结构比较重要,过大的网络容易过拟合，虽然前期训练块，但换对手就容易卡住
    - `512 * 4`对Actor网络似乎够用，但对Critic网络不够用（从loss上看policy loss可以收敛，而value loss不行，虽然也可能是存在其他的问题），目前Mlagent的Actor与Critic是相同结构的，可以考虑扩大后者。
    - `768 * 3`以及`768 * 4`，前期reward上升块，但进度1/5后基本就卡住了，无法适应新对手，这时候虽然value loss有收敛，但policy loss无法收敛
    - `512 * 3`应该是过小了，连路都不会走，不过如果只让它学习寻路，不去控制炮塔应该可以
2. 奖励函数最重要，决定了你想让AI按什么策略行动，密集的奖励能让模型更容易学习，但限制了它自由发挥的空间。可以考虑在训练后期调整奖励函数，以得分为最终奖励
    - 但我的奖励函数显然还有问题，模型无法学会正确的瞄准与开火，导致其他能力也受影响，只能花最后一天亡羊补牢
    - 也可能是观察空间，输出空间或者动作映射有问题，模型无法正确地处理弹道，炮管旋转等信息
    - 观察空间一开始为了适应其他任务做大了，可能专注于1V1比较好
    - 按GPT的意见增加了堆叠输入，没感觉到效果
    - 规则AI的命中率能达到0.8，训练结果只有0.1，仅供参考，好让后来者改进
3. 课程学习有助于模型适应新对手，可以通过调整对手池的强弱分布来控制难度，另外对手的多样性也很重要，只提供RuleBasedAI这一个对手无法提高它面对其他对手时的表现。平滑的难度过渡也有用
    - 行为克隆（BC）消耗总步数的1/50即可，可以把强度调大点
    - `time_horizon: 1024`应该是最为合适的长度，一局的长度在`DecisionInterval = 5或3`时只有1800步，但`= 1`时能达到8000步，过小的间隔带来更多的噪声，不好让模型收敛，学习率和 batch_size 可以在默认基础上适当调整
4. 目前一轮训练（100M）需要70小时+，我个人配置为`5080laptop +  32G内存`，可以同时开6个并行，占用内存10G+，显卡占用率能达到100%。在一轮训练前需要仔细检查，不要等炼一半了才发现配置有问题，mlagent还是有bug的，resume训练会重置BC的计数器，过强的BC可能会使训练崩溃。
5. 看到模型reward涨不动不要心存侥幸，基本上配置和奖励函数还是有问题。
6. 训练完记得多看看实战表现，不看你不知道它的炮管根本喵不准，evaluation里体现不出来
    - 我在最后检查时才发现模型不会处理敌人死亡时的情况，得加个没有敌人的课程
    - 同理，模型没在B队上训练过，效果也不会好
7. 目前AI对强化学习的理解还是不太行，GPT5.5效果可以，但很消耗额度（30+美元）。而K2.6虽然可以辅助修改，但缺乏关于MLagent的正确知识，在它给的参数建议上踩了很多坑。
    - 有AI偷偷改了核心类代码，迁移后才发现。虽然消除了编译报错，但TrainingMatchController和TankBattleActionMapper可能会出问题。
8. 想不到了
